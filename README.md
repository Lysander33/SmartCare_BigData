# 智慧医养大数据分析项目

这是我在大三期间完成的实训项目，基于 **Hadoop + Spark + Hive** 架构，旨在解决老龄化背景下养老机构的资源调配与健康预测问题。

##  技术栈
* **平台**: Hadoop 3.3.5 (三节点集群), Hive 3.1.3
* **计算**: Spark 3.5.1 (PySpark)
* **环境**: CentOS 9, Python 3.9
* **可视化**: Seaborn, Matplotlib

##  核心文件说明
* `etl_process.py`: 负责从 Hive 读取原始数据，进行年龄过滤、缺失值填充等清洗工作。
* `analysis_visual.py`: 实现床位利用率趋势、服务热度等多维度可视化洞察。
* `setup_cluster.sh`: 记录了整个大数据集群的环境配置流程。
* `智慧医养分析报告.pdf`: 包含详细的业务洞察与项目总结。

##  核心洞察
通过分析识别出医疗护理为高频需求，并发现了床位利用率的季节性波动规律，为机构提供了数据驱动的运营建议。

##  项目难点与反思
* **环境兼容性挑战**：在集成 Spark 与 Hive 时，遇到了经典的元数据版本冲突问题。通过查阅 Apache 官方文档，定位到 `spark.sql.hive.metastore.jars` 的配置路径问题并成功修复。这次经历让我对大数据组件间的通信机制有了更深的理解。
